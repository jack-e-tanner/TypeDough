There are two architectures living in this codebase. They are independent —
the editor mutates the graph; the evaluator runs it.

  1. Editor Architecture (MVC)  - how the user builds a graph in the UI
  2. Evaluation Architecture    - how a finished graph computes outputs

1. Editor Architecture  (Model - View - Controller)

                       ┌─────────────────────────────┐
                       │         MainWindow          │
                       │   (chrome, menus, layout)   │
                       └──────────────┬──────────────┘
                                      │ owns
                                      ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │                          GraphEditor                             │
   │                         (CONTROLLER)                             │
   │                                                                  │
   │   - owns the QGraphicsScene                                      │
   │   - holds a pointer to GraphManager                              │
   │   - translates view events  → model calls                        │
   │   - reacts to model signals → spawns/destroys view items         │
   └─────────────────┬──────────────────────────────┬─────────────────┘
                     │                              │
               user events ▲                  change signals ▲
                     │     │                        │
              spawn / kill ▼                 mutate state ▼
                     │                              │
                     ▼                              ▼
   ┌────────────────────────────────┐  ┌────────────────────────────────┐
   │         QGraphicsScene         │  │          GraphManager          │
   │             (VIEW)             │  │            (MODEL)             │
   │                                │  │                                │
   │   NodeItem    WireItem         │  │   nodes (id → Node)            │
   │   PortItem                     │  │   connections (incoming)       │
   │   hover / drag state           │  │   cached outputs               │
   │                                │  │   emits signals on change      │
   └────────────────────────────────┘  └────────────────────────────────┘

* The MODEL never touches QGraphicsScene, NodeItem, PortItem, WireItem,
  QPointF, or anything else from QtWidgets. It deals in node IDs,
  logical ports, and connections only.

* The VIEW never calls GraphManager directly. It emits Qt signals
  ("user dragged from this port to that one") and lets the controller
  decide what to do.

* The CONTROLLER is the ONLY layer that touches both sides. It listens
  to view events and model signals, and is the place where mutations
  get committed.


2. Evaluation Architecture  (pull-based, topo-sorted)

                ┌──────────────────────────────────────┐
                │             GraphManager             │
                │                                      │
                │   1. topological sort                │
                │   2. for each node in order:         │
                │        a. construct a NodeRuntime    │
                │        b. node.evaluate(runtime)     │
                │        c. cache the output           │
                └────────────────┬─────────────────────┘
                                 │  constructs per node, on stack
                                 ▼
                ┌──────────────────────────────────────┐
                │             NodeRuntime              │
                │     (transient evaluation context)   │
                │                                      │
                │   - pulls inputs from the manager's  │
                │     cache of upstream outputs        │
                │   - writes this node's output back   │
                │   - destroyed at end of evaluate()   │
                └────────────────┬─────────────────────┘
                                 │  passed by reference
                                 ▼
                ┌──────────────────────────────────────┐
                │                 Node                 │
                │       (AddNode, SubNode, ...)        │
                │                                      │
                │   evaluate(runtime):                 │
                │     a = runtime.input(0)             │
                │     b = runtime.input(1)             │
                │     runtime.set_output(0, a + b)     │
                └──────────────────────────────────────┘
